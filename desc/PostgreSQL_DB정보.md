# 서울 지하철 공공 데이터 DB 정보

스키마 이름은 소문자로 사용한다. *(대문자는 쿼리 사용시, 쌍따옴표를 붙여야하는 불편함을 최소화 하기 위함. PostgreSQL DBMS 특성.)*

## ODS (Operational Data Source) 스키마
- `ods` 스키마 테이블들은 원천데이터를 최대한 가공없이, 그대로 저장 보관한다.  
*( 외부 데이터를 그대로 옮긴 보관용이여서 "Staging Area"에 가깝지만, 임시 저장용은 아니라 <u>추후에도 계속 사용하기 위해 데이터 보관 관리</u>가 계속 진행되어야 하기에 'ods'명이 더 적절하다고 판단함. )*
- 테이블 네이밍 규칙 : `{원천 API 시스템명}`_`{API명 또는 데이터 유추 가능한 이름}`


## 📊 서울 열린데이터 광장 데이터 수집 명세 

### 🏛️ 1. 데이터 수집 및 업데이트 주기 (Real-time Spec)
- **일별 데이터 (`CardSubwayStatsNew`)**: 전월/전일 데이터가 매일 새벽 대략 2~3일의 데이터 시차(Lag)를 두고 순차적으로 누적 업데이트됨.
- **시간대별 데이터 (`CardSubwayTimeInfo`)**: 매월 확정 데이터가 익월(다음 달) 3일 ~ 10일 사이에 일괄 확정 및 업데이트됨.  
  

> 시간대별 월별 데이터 (CardSubwayTimeInfo): 당월 데이터 익월(다음 달) 3일에서 10일 사이에 업데이트. 
> 
> - API 목록 : [👉 서울열린데이터광장 전체 데이터셋 목록 URL  ](https://data.seoul.go.kr/dataList/datasetList.do)
> - API 인증키 : `새로발급필요`
