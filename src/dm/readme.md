# 🚇 서울시 지하철 교통약자 인프라 과부하 지수 Data Mart 구축

> **PostgreSQL을 활용하여 서울시 지하철역의 교통약자 편의시설(엘리베이터) 수요 대비 공급 불균형을 추적하는 데이터 마트 모델링 프로젝트입니다.**

## 🏗️ 1. 아키텍처 핵심 요약
* **Fan-out(데이터 뻥튀기) 대참사 방지**: 환승역(ex: 시청, 홍대입구) 조인 시 데이터가 중복 복사되는 리스크를 **다중 CTE(WITH 절) 사전 집계 기술**로 완벽 차단.
* **질적 지표로의 전환**: 단순히 이용객이 많은 역이 아닌, 엘리베이터 1대당 부담 인원을 계산하여 **진짜 인프라 사각지대(ex: 까치산역, 낙성대역)**를 우선순위로 도출.

## 🛠️ 2. 사용된 핵심 SQL 기술
* **Multi-CTE (다중 WITH 절)**: 공정별 가공 단계를 분리하여 메인 쿼리 가독성 극대화.
* **Window Function**: `DENSE_RANK() OVER (PARTITION BY ... ORDER BY ... DESC NULLS FIRST)`를 사용하여 엘리베이터가 0대인 위험 구역을 최상위 순위로 자동 제어.
