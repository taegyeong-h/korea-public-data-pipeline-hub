# 🚲 서울시 공공자전거(따릉이) 이용 현황 파이프라인

본 파이프라인은 서울열린데이터광장의 데이터를 활용하여, **[서울시 자치구별 공공자전거 대여소 가용성 및 이용 패턴 분석 마트]**를 구축하기 위한 데이터 정의 및 엔지니어링 수집 전략을 기록합니다.

## 1. 기회 의도 및 데이터 선정 사유
서울시 전역의 자전거 대여소 데이터를 다각도로 분석하여, **"어느 자치구의 대여소가 가장 붐비는가?"** 및 **"자전거 부족 현상이 발생하는 시간대는 언제인가?"**라는 비즈니스 가설을 검증하고자 합니다.

## 2. 데이터 모델링 전략 (Star Schema)
데이터의 성격에 맞춰 **기준 데이터(Dimension)**와 **실적 데이터(Fact)**를 이원화하여 관리합니다.

### A. 기준 데이터 (Dimension Table)
* **대상**: 서울시 공공자전거 대여소 정보
* **파일**: `src/ODS/seed_data/seoul_bike_station.csv`
* **역할**: 대여소 번호에 따른 자치구, 위치 정보(위도/경도) 매핑
* **수집 방식**: 웹 포털에서 최신 CSV 파일을 최초 1회 다운로드하여 적재(Full Refresh)

### B. 실적 데이터 (Fact Table)
* **대상**: 서울특별시 공공자전거 실시간 대여정보
* **스크립트**: `src/ODS/extract_fact_bike_daily.py`
* **역할**: 대여소별 매시간 '현재 대여 가능한 자전거 수' 추적
* **수집 방식**: 파이썬을 활용해 매일 새벽 API를 호출하여 데이터 레이크에 누적(Batch Fetch)

## 3. 수집 대상 데이터셋 명세
| 데이터 분류 | 데이터명 | 파일/스크립트명 | 주기 | 포맷 |
| :--- | :--- | :--- | :--- | :--- |
| **Dimension** | 서울시 공공자전거 대여소 현황 | `seoul_bike_station.csv` | 초기 1회 | CSV |
| **Fact** | 공공자전거 실시간 대여정보 | `extract_fact_bike_daily.py` | 매일 새벽 | API(JSON) |
