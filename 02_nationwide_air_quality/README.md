# 🌬️ 전국 대기질(미세먼지) 행정구역별 오염도 파이프라인

본 문서는 공공데이터포털(data.go.kr)의 한국환경공단 에어코리아 데이터를 활용하여, [대한민국 전 영토의 행정구역별 일별 미세먼지 오염도 랭킹 분석 마트]를 구축하기 위한 데이터 정의 및 엔지니어링 수집 전략을 기록합니다.

---

## 🎯 1. 기획 의도 및 데이터 선정 사유
대한민국 전체 시도 및 시군구 단위의 대기오염 수치를 다각도로 분석하여, **"전국에서 가장 숨쉬기 힘든 최악의 대기질 오염 지역 TOP 5 자치구는 어디인가?"** 라는 비즈니스 가설을 검증하고자 합니다. 이를 위해 데이터의 성격에 맞춰 수집 데이터셋을 이원화(Fact & Dimension) 설계했습니다.

---

## 🗂️ 2. 데이터 모델링 및 소스 수집 전략 (Star Schema)
본 파이프라인은 인프라 안정성과 API 트래픽 최적화를 위해 변하지 않는 **기준 정보(Dimension)** 는 내부 시드 파일로 품고, 매일 쌓이는 **실적 데이터(Fact)** 는 배치 API로 긁어오는 무결점 스타 스키마 구조를 지향합니다.

### 🏢 A. 마스터 프로필 데이터 (Dimension Table)
* **대상 데이터**: 전국 대기오염 측정소 현황 정보
* **파일명**: `src/ODS/seed_date/nationwide_air_station.csv`
* **선정 사유**: 실적 팩트 테이블에 존재하는 단순 '측정소명'에 `시도명(ex: 경기도)`, `시군구명(ex: 수원시)`이라는 행정구역 속성을 결합하여 전국 단위의 공간 분석 외연을 확장하기 위함입니다.
* **수집 및 파이프라인 방식 (Seed Load)**: 측정소 위치나 행정구역 정보는 새 측정소가 신설되지 않는 한 거의 변하지 않는 준정적(Quasi-Static) 특성을 가집니다. 이를 매일 API로 중복 호출하는 것은 심각한 자원 낭비이므로, **웹 포털에서 최신 CSV 파일을 최초 1회 다운로드하여 `seed_date/` 폴더에 내장한 뒤 GCS로 적재(Full Refresh)**하는 고효율 아키텍처를 채택합니다.

### 📊 B. 비즈니스 실적 데이터 (Fact Table)
* **대상 데이터**: 한국환경공단 에어코리아 대기오염정보 조회 서비스 (`측정소별 실시간 측정정보 조회`)
* **스크립트**: `src/ODS/extract_fact_air_quality_daily.py`
* **선정 사유**: 전국 측정소별 매시간/매일 찍히는 미세먼지(PM10), 초미세먼지(PM2.5) 확정 수치를 추적하여 분석 마트의 핵심 계량 지표(Measure)로 활용합니다.
* **수집 및 파이프라인 방식 (Batch Fetch)**: 해당 데이터는 매일 실시간으로 누적 업데이트되는 트랜잭션 데이터입니다. 사람이 수동으로 관리할 수 없으므로, **파이썬 스크립트를 활용해 원격지 API 서버에서 데이터를 실시간으로 Fetch 하여 GCS 데이터 레이크로 직송하는 데일리 배치 파이프라인**을 구축합니다.
