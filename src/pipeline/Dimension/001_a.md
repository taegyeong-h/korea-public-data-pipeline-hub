# 클라이언트 사이드 방식 (DBeaver의 '데이터 가져오기' / 파이썬 / psql \copy)

지금 화면에 엑셀(CSV)을 띄워놓으신 PC가 호스트 OS(내 컴퓨터)이고, DBeaver도 거기에 설치되어 있다면 게스트 OS에 파일을 옮길 필요가 전혀 없습니다.

DBeaver에서 192.168.56.10으로 접속합니다.

앞서 만든 ods.tm_subway_station_master 테이블을 마우스 우클릭합니다.


엑셀파일 CSV UTF-8(쉼표로 분리)(*.csv) 로 변환

[데이터 가져오기 (Import Data)]를 누릅니다.

이때 탐색기 창이 뜨면, 호스트 OS(내 컴퓨터)에 있는 CSV 파일을 선택해 줍니다.

DBeaver가 알아서 내 컴퓨터의 CSV를 읽어 네트워크를 통해 VirtualBox 안의 Postgres로 초고속 주입을 해줍니다.

한글이 안 깨지고 아주 깨끗하게 나오는 것은 100% 완벽하게 성공하신 게 맞습니다! 인코딩 문제는 완전히 해결되었습니다.

하지만, 이 상태로 바로 [진행]을 누르시면 안 됩니다! ⚠️

지금 Target 컬럼을 보시면 우리가 설계한 영어 컬럼명(station_cd, station_nm 등)이 아니라 한글 컬럼명들이 들어가 있고, Mapping 칸에도 create(새로 만들겠다)라고 되어 있습니다.

이대로 진행하면 우리가 정성껏 만든 기존 테이블에 한글 이름으로 된 기차 칸(컬럼)들이 새로 추가되어 꼬이게 됩니다.

우리가 이미 만들어둔 영어 컬럼방에 데이터를 쏙쏙 집어넣으려면 딱 한 번만 마우스로 짝꿍(매핑)을 수동으로 맞춰주셔야 합니다.

🛠️ 마지막 매핑 수정 방법 (이렇게 바꿔주세요!)
화면 중앙에 있는 표의 Target 열에 있는 글자들을 마우스로 하나씩 클릭해 보세요. 드롭다운(선택창)이 열리면서 우리가 만든 영어 컬럼 목록이 뜰 것입니다. 아래와 같이 매칭시켜 줍니다.

역사_ID 줄의 Target 칸을 클릭 ➔ station_cd 선택

역사명 줄의 Target 칸을 클릭 ➔ station_nm 선택

호선 줄의 Target 칸을 클릭 ➔ route_ln_nm 선택

위도 줄의 Target 칸을 클릭 ➔ latitude 선택

경도 줄의 Target 칸을 클릭 ➔ longitude 선택

(이렇게 선택하고 나면 Mapping 칸의 글자도 create에서 existing(기존 컬럼 사용) 또는 정상 매핑 상태로 바뀔 것입니다.)

🚀 다 바꾸셨다면?
5개 컬럼의 Target을 모두 영어 컬럼명으로 매칭 완료하신 후에 하단의 [다음(N) >]을 누르고 [진행(P)]을 누르시면 데이터베이스 적재가 완벽무결하게 완료됩니다!

마지막 관문입니다. 짝꿍만 영어로 바꿔서 바로 진행해 보세요!
